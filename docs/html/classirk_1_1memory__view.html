Detailed Description

A base class for all types of memory views.

A memory view is an abstraction for accessing any memory area, be it in main memory, or on disk, or via a memory mapped file. The constructor accepts different types of memory sources that implement memory access details. These types are hidden with type erasure: only the constructor is a template, and memory_view type can be used polymorphically.

const char* irk::memory_view::data ( ) const
inline

Returns a pointer to the underlying data.

The source is expected to return a pointer of contiguous data in memory (or memory mapped file). Thus, if a source loads data lazily, it must first load the entire data and return its pointer. To avoid such inefficiency, e.g., if data is meant to be accessed sequentially, use begin() and end(), or operator[] instead.
